# Канонические примеры — `tests-store`

В этой папке должны лежать реальные примеры из проекта, на которые скилл будет опираться. Сейчас их нет — нужно добавить.

## Что положить

Минимум одну пару «стор + тест к нему». В идеале — два разных стора, чтобы покрыть распространённые случаи.

### Обязательная пара (любой простой стор)

- `<storeName>.ts` — сам zustand-стор. Подойдёт стор с 2–4 actions и 1–2 селекторами. Например, фильтры списка, выбранные строки таблицы, открытые модалки.
- `<storeName>.test.ts` — тест к нему. Должен показывать:
  - сохранение `initialState` до `describe`,
  - сброс через `setState(initialState, true)` в `beforeEach`,
  - один `describe` на каждое action,
  - покрытие начального состояния, всех actions и хотя бы одного селектора.

### Желательно ещё (для разнообразия)

Стор с одним из дополнительных паттернов, если такой есть в проекте:

- стор с `persist` middleware — пример сброса хранилища в `beforeEach`;
- стор с производным селектором (вычисляется из других полей) — пример теста на разные стейты;
- стор с асинхронным action (`async set`) — пример с `await act(...)`.

## Как сослаться из SKILL.md

Раздел `Канонические примеры` уже добавлен в `../SKILL.md` и говорит «следуй файлам из `./examples/`». Когда положишь файлы — никакого дополнительного редактирования скилла не нужно, qwen подхватит их автоматически.

## Что не нужно

- Не клади «тестовые проекты целиком» — только один стор + один тест.
- Не клади примеры с моками компонентов, читающих стор. Это другой случай — он лежит в `tests-components-mocked/examples/`.
