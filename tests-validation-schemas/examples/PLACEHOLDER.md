# Канонические примеры — `tests-validation-schemas`

В этой папке должны лежать реальные примеры из проекта, на которые скилл будет опираться. Сейчас их нет — нужно добавить.

## Что положить

Минимум одну пару «схема + тест». В идеале — две: одна на yup, одна на zod, потому что в `package.json` есть оба.

### Обязательная пара (yup-схема)

Идеальный кандидат — `validation-schema.ts` из `DocumentsSidebar` (`getValidationSchema(calendarData)`):

- `validation-schema.ts` — фабрика, возвращающая yup-схему, зависящая от `calendarData`.
- `validation-schema.test.ts` — тест к ней. Должен показывать:
  - блок `describe('успешная валидация')` с `validate(...)` / `isValid(...)`,
  - блок `describe('ошибки валидации')` с `validate(...)` через `rejects.toThrow(...)`,
  - тест с `abortEarly: false` для проверки нескольких ошибок одновременно,
  - тесты на динамику от `calendarData` (срабатывает / не срабатывает кросс-полевая проверка).

### Желательно ещё (zod)

- `<entity>.schema.ts` — zod-схема (например, ответ API).
- `<entity>.schema.test.ts` — тест на `parse` / `safeParse`, проверка `path` и `code` в `error.issues`.

## Как сослаться из SKILL.md

Раздел `Канонические примеры` уже добавлен в `../SKILL.md` и ссылается на эту папку. Когда положишь файлы — qwen подхватит их автоматически.

## Что не нужно

- Не клади тесты форм, которые применяют схему — они в `tests-formik-form/examples/`.
- Не клади весь набор схем проекта целиком — двух достаточно для шаблона.
