# Канонические примеры — `tests-hooks-generic`

В этой папке должны лежать реальные примеры из проекта, на которые скилл будет опираться. Сейчас их нет — нужно добавить.

## Что положить

Минимум одну пару «хук + тест». В идеале — три, чтобы покрыть основные паттерны.

### Обязательная пара (хук состояния)

- `useToggle.ts` (или другой простой хук на `useState`/`useCallback`) — небольшой хук, возвращающий пару `[value, action]`.
- `useToggle.test.ts` — тест к нему. Должен показывать:
  - `renderHook` из `@testing-library/react-hooks`,
  - проверку начального значения,
  - вызов action через `act(() => result.current[1]())`,
  - проверку `initialProps`.

### Желательно ещё

- **Хук с таймером** (`useDebouncedValue`, `useThrottle`) + тест — пример с `jest.useFakeTimers()` / `jest.advanceTimersByTime()` в `act`.
- **Хук с подпиской на DOM** (`useOnClickOutside`, `useEventListener`) + тест — пример с `dispatchEvent`, проверкой `unmount` и снятия подписки.

## Как сослаться из SKILL.md

Раздел `Канонические примеры` уже добавлен в `../SKILL.md` и ссылается на эту папку. Когда положишь файлы — qwen подхватит их автоматически.

## Что не нужно

- Не клади хуки, которые ходят в сеть — они в `tests-queries/examples/`.
- Не клади хуки, читающие zustand-стор — там либо `tests-store/examples/` (если тестируется сам стор), либо мок стора в тесте компонента.
- Не клади хуки, использующие `react-router-dom` — для них `tests-routes/examples/`.
