# Канонические примеры — `tests-components`

При написании теста на UI-компонент сначала смотри сюда, потом — в `../SKILL.md`. Если стиль расходится — приоритет у файлов в этой папке.

## Файлы

- `CalendarField.tsx` — пример компонента: тонкая обёртка, выбирающая `FormikCalendar` или `BasicCalendar` по наличию `name`.
- `CalendarField.test.tsx` — эталонный тест: рендер в `<Formik>`, `userEvent.setup()`, выборка элементов по `data-testid`, асинхронные действия пользователя.

> Файлы — копия рабочего компонента из проекта (`features/.../CalendarField`). При переносе в репозиторий замени их на актуальную версию.
