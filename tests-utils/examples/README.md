# Канонические примеры — `tests-utils`

При написании теста на чистую функцию следуй сначала этим примерам, и только потом — абстрактным шаблонам в `../SKILL.md`. Если стиль расходится — приоритет у файлов в этой папке.

## Файлы

- `calendar.ts` — пример исходника (несколько утилит на `date-fns`).
- `calendar.test.ts` — эталонный набор тестов: один `describe` на функцию, набор `it` на happy path и edge-cases.

> Эти файлы — копия рабочих утилит из проекта (модуль `@shared/utils/functions/calendar`). При переносе в репозиторий замени их на актуальную версию из `master`.
