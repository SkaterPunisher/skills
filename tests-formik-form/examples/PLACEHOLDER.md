# Канонические примеры — `tests-formik-form`

В этой папке должны лежать реальные примеры из проекта, на которые скилл будет опираться. Сейчас их нет — нужно добавить.

## Что положить

Минимум одну пару «форма + тест». Желательно — две, чтобы покрыть простую и сложную форму.

### Обязательная пара (простая форма)

- `<FormName>.tsx` — небольшая форма Formik (3–5 полей), с `validationSchema` (yup или zod) и кнопками submit/reset. Хороший кандидат — фильтры списка, простая модалка создания записи.
- `<FormName>.test.tsx` — тест к ней. Должен показывать:
  - `userEvent.setup()` на верхнем уровне `describe`,
  - happy submit: ввод валидных данных → клик по submit → `await waitFor(() => expect(handleSubmit).toHaveBeenCalledWith(...))`,
  - ошибка валидации: попытка submit без обязательных полей → `findByText(/обязательное/)`, `expect(handleSubmit).not.toHaveBeenCalled()`,
  - reset: клик по кнопке reset → поля очищены до `initialValues`,
  - дизейбл submit (если есть): `expect(submitBtn).toBeDisabled()` до заполнения, `not.toBeDisabled()` после.

### Желательно ещё (сложная форма)

- Форма с `enableReinitialize`, кросс-полевыми правилами (например, диапазон дат `start < end`) или полями, зависящими от значения других — и тест на эти случаи.

## Как сослаться из SKILL.md

Раздел `Канонические примеры` уже добавлен в `../SKILL.md` и ссылается на эту папку. Когда положишь файлы — qwen подхватит их автоматически.

## Что не нужно

- Не клади тесты на отдельные поля Formik (например, `CalendarField`) — они уже лежат в `tests-components/examples/`.
- Не клади тесты только на схему валидации — они в `tests-validation-schemas/examples/`.
